Herramientas de KitZ (OS)

124+ herramientas invocables vía el gateway de KitZ OS. Autenticación JWT, headers seguros y respuestas JSON.

Core

Health Check — Estado del Sistema

GET /api/kitz/status Authorization: Bearer <JWT> // Respuesta: { "status": "ok", "tools": 124, "agents": 33, "uptime": "72h 14m" }
Core

tool-calls — Invocar Herramienta

POST /tool-calls Authorization: Bearer <JWT> x-org-id: org-123 Content-Type: application/json { "name": "gmail_send", "input": { "to": "cliente@ejemplo.com", "subject": "Propuesta comercial", "body": "Adjunto la propuesta..." }, "riskLevel": "low", "requiredScopes": ["tools:invoke"] }

Principios del proyecto

01
Zero-Trust Security

JWT + RBAC + scopes en cada request. Rate limiting, headers seguros y aislamiento multi-tenant por defecto.

02
Modular por diseño

Cada tool es independiente. Agrega, modifica o elimina herramientas sin afectar el resto del sistema.

03
LATAM first

Construido para y por la comunidad latinoamericana. Documentación en español, ejemplos regionales.

04
AI-native — 33 Agentes

33 agentes de IA (C-suite, Board, Governance) con 124+ tools. Claude, OpenAI, Gemini, Perplexity y DeepSeek integrados.

05
Código abierto, siempre

El conocimiento y las herramientas deben ser accesibles para todos. Sin muros de pago, sin restricciones.

06
Draft-first

Todos los mensajes salientes (email, WhatsApp, SMS) pasan por revisión antes de enviarse. Seguridad ante todo.
