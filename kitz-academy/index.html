Herramientas del KitZ Gateway

Ejemplos reales de las herramientas disponibles en el gateway. Copia los ejemplos JSON y prueba localmente.

Core

echo — Prueba de Conectividad

POST /v0.1/tools/echo Content-Type: application/json { "message": "hola desde KitZ" } // Respuesta: { "result": "hola desde KitZ" }
Core

utc_now — Timestamp UTC

POST /v0.1/tools/utc_now Content-Type: application/json {} // Respuesta: { "result": "2026-03-14T10:30:00Z" }
Core

sum — Sumar Valores

POST /v0.1/tools/sum Content-Type: application/json { "values": [10, 25, 37, 8] } // Respuesta: { "result": 80 }
Datos

list_assets — Consultar Activos

POST /v0.1/tools/list_assets Content-Type: application/json { "client": "Acme Corp", "status": "active" } // Filtra por cliente, tier, status, // marca o fecha de vencimiento
Datos

get_asset_metrics — Métricas del Portafolio

POST /v0.1/tools/get_asset_metrics Content-Type: application/json {} // Respuesta incluye: // total_devices, unique_clients, // oem_cost, tpm_cost, savings, alerts
API

create_order — Crear Orden de Compra

POST /v0.1/tools/create_order Content-Type: application/json { "client": "TechLatam SA", "items": ["asset_001", "asset_002"], "type": "tpm" } // Crea PO en estado "draft"
API

list_orders — Consultar Órdenes

POST /v0.1/tools/list_orders Content-Type: application/json { "status": "pending-approval" } // Filtra por estado del flujo: // draft, pending-approval, approved, // submitted, fulfilled
IA

generate_insights — Análisis con IA

POST /v0.1/tools/generate_insights Content-Type: application/json {} // Genera automáticamente: // - Concentración de clientes // - Heatmap de vencimientos // - Comparativa de costos // - Revenue at risk
API

Listar Tools Disponibles (GET)

GET /v0.1/tools // Respuesta: lista de todas las // herramientas registradas en el // gateway con sus nombres y // descripciones. // No requiere body.
Core

Health Check

GET /v0.1/health // Respuesta: { "status": "ok", "version": "0.1" } // Verifica que el gateway // esta corriendo correctamente

Principios del proyecto

01
Zero dependencias

El gateway usa exclusivamente la biblioteca estándar de Python. Sin pip install, sin node_modules, sin sorpresas.

02
Modular por diseño

Cada tool es independiente. Agrega, modifica o elimina herramientas sin afectar el resto del sistema.

03
LATAM first

Construido para y por la comunidad latinoamericana. Documentación en español, ejemplos regionales.

04
AI-native

Diseñado para integrarse con modelos de lenguaje como Claude. Los tools son invocables por agentes de IA.

05
Código abierto, siempre

El conocimiento y las herramientas deben ser accesibles para todos. Sin muros de pago, sin restricciones.

06
Ligero y portable

Corre en cualquier máquina con Python 3. Un solo archivo, un solo comando para iniciar.
